실험 AWS MQTT client max

AWS lightsail 에 설치된 Mosquitto 브로커의 최대 허용 클라이언트 수를 테스트해봤다.

Macbook 에서 다수의 MQTT.JS Node client 를 생성했다.

let { Command } = require('commander')
var mqtt = require('mqtt')
let intervalID = null;

let program  = new Command()
let clients = [];
program.option('-n, --number-Clients <number>','number of client instance')
program.parse()

let opt = program.opts();

function onMessage( topic ,message){
  console.log("\n##### topic:"+ topic);
  console.log("playloadString:"+message.toString());
}
  
function onError(e){
  if(e) console.log('onError', e)
}

if(opt.numberClients >= 1){
  console.log('N', opt.numberClients )
  for(let i = 0 ; i < opt.numberClients  ; i++){
     let c = mqtt.connect('mqtt://congtrol.com');
      c.on('connect', e=>{
        c.subscribe('#', onError); } );
      c.on('message', onMessage)

      clients =  [ ...clients, c];
      console.log('len: ',clients.length)
  }
 
}
$ node pubsub-multiple.js -n 100

결과1. 최대 접속수가 1000을 넘자 모스키토가 서비스 불능상태가 됨.

파일개수 최대개수가 1000으로 지정되어있다.

임시로 설정하면 그후 실행시 일시 반영되는것 같다.

$ ulimit -n 2000

하지만 시스템 전반 반영하려면 설정파일을 수정해줘야한다.

root 의 경우 도메인을 root 로 명시해야한다.

온라인 문서상 여러내용이 나오고 여러 파일의 수정이 안내되어있었다

/etc/systemd/system.conf ?

그런데 아래 파일만 수정해도 작동됨을 확인함.

$ sudo vi /etc/security/limits.conf

# 아래 두줄 추가
bitnami soft nofile 5000
bitnami hard nofile 10000

도메인:  id, group 등
soft: 경고 수준
hard: 실제 한계

이후, 2500 연결 정도는 작동되었다.

문제는 CPU 5%수준이 600접속 정도에서 5% 도달.
